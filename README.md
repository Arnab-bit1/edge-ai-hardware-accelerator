> **A highly optimized, RTL-level 1D Convolutional Neural Network accelerator designed for Edge AI, featuring a 100% hand-coded Multiply-Accumulate (MAC) datapath.**

## 📌 The Architecture Breakdown
This repository contains the digital design and verification of a hardware accelerator built for low-latency sequential data processing. Instead of relying on standard vendor IP blocks, the core arithmetic logic was architected from the ground up to maximize throughput and minimize area overhead on resource-constrained FPGAs.

## 📁 Approach
* **Dataset used:** MIT-BIH Arrhythmia Database [https://physionet.org/content/mitdb/1.0.0/]
* **Initial training** TensorFlow to obtain weights and biases
* **Designing Custom Hardware** Designed complete datapath (RAM, MAC Unit, ReLU Unit, MaxPooling) and control unit.

### Core Features
* **Custom MAC Datapath:** Fully hand-coded Multiply-Accumulate units tailored for specific bit-widths, bypassing generic DSP inference.
* **Sliding Window Architecture:** Pipelined control logic optimized for real-time 1D signal convolutions.
* **Edge-Optimized:** Architected specifically for high-efficiency FPGA deployment.

## ⚙️ Working
The accelerator is trained on weights and biases generated by Tensorflow on the MIT-BIH Arrythmia Dataset. The accelerator takes a 256 byte input sample of heartbeat readings and on the basis of weights and biases, gives an arrythmia prediction. It has been verified using 2 types of inputs - both generated by the Tensorflow trained model.

## 📁 Repository Structure
* `/rtl`: Modularized synthesizable Verilog (`mac_unit.v`, `conv1d_layer.v`, `accelerator_top.v`).
* `/tb`: Verification environment and stimulus generation.
* `/docs`: Project reports, architectural diagrams, and timing constraints.
* `/jupyter_notebook`: Model training notebook
* `/input_data`: Contains 2 input files, one for sick patient and another for healthy patient
* `/weights_and_biases`: Contains the weights and bias values for each layer. These values were used in the accelerator
